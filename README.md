# Спортивная статистика - Веб-приложение

Современное веб-приложение для отображения спортивной статистики, результатов матчей и коэффициентов.

## 🚀 Технологический стек

- **Next.js 16** (App Router)
- **React 18+**
- **TypeScript**
- **Tailwind CSS**
- **TanStack Query (React Query)**
- **Axios**
- **lucide-react** (иконки)
- **date-fns** (работа с датами)

## 📦 Установка

```bash
npm install
```

## 🏃 Запуск

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

## 🔌 Интеграция с реальным API

### Шаг 1: Проверка API

Приложение настроено на работу с API `https://api.sstats.net/v1`. Проверьте доступность:

```bash
curl https://api.sstats.net/v1/matches
```

### Шаг 2: Обновление типов

Когда получите реальный JSON-ответ от API, обновите типы в `lib/types.ts`:

```typescript
// Пример структуры, которую нужно адаптировать под реальный API
export interface Match {
  id: string
  sport_type: string
  league_name: string
  team1: Team
  team2: Team
  start_time: string
  is_live: boolean
  score?: Score
  odds: Odds
}
```

### Шаг 3: Замена mock-данных на реальные

Замените содержимое `app/page.tsx` на код из `app/page-with-api.tsx.example`:

```bash
# Windows PowerShell
Copy-Item app/page-with-api.tsx.example app/page.tsx
```

### Шаг 4: Настройка CORS

CORS уже настроен в `next.config.ts`:

```typescript
async rewrites() {
  return [
    {
      source: '/api/proxy/:path*',
      destination: 'https://api.sstats.net/v1/:path*',
    },
  ]
}
```

Все запросы к `/api/proxy/*` будут проксироваться на `api.sstats.net`.

## 🎨 Основные компоненты

### Header
Липкая шапка с навигацией, поиском и кнопками авторизации.

### BannerCarousel
Горизонтальная карусель с крупными баннерами (Live, Лига Чемпионов и т.д.).

### SportFilter
Фильтр по видам спорта с иконками (Футбол, Хоккей, Теннис и т.д.).

### MatchCard
Карточка матча с:
- Тегом вида спорта
- Названием турнира
- Логотипами и названиями команд
- Временем начала или Live-индикатором
- Счетом (если матч идет)
- Коэффициентами (П1, X, П2)

## 🔄 Работа с данными

### Mock-данные (текущая версия)

Используются данные из `lib/mock-data.ts`. Идеально для разработки UI без зависимости от API.

### Реальное API

Хук `useMatches` автоматически:
- Делает запросы к API
- Кэширует результаты
- Обновляет данные каждые 30 секунд для Live-матчей
- Обрабатывает состояния загрузки и ошибок

```typescript
const { data, isLoading, error } = useMatches({
  sport: 'football',
  is_live: true,
})
```

## 📝 Примечания

- **Логотипы команд**: Сейчас используются заглушки. Когда API вернет URL логотипов, они автоматически отобразятся.
- **Коэффициенты**: Для теннис/баскетбол коэффициент X = 0 (нет ничьей), карточка автоматически адаптируется.
- **Время**: Все даты форматируются с учетом локали (ru).

## 🐛 Отладка

### Проблемы с CORS
Если видите ошибки CORS в консоли браузера, убедитесь что:
1. Next.js dev-сервер запущен
2. Запросы идут через `/api/proxy/`, а не напрямую к `api.sstats.net`

### Проблемы с типами
Если TypeScript ругается на типы данных от API:
1. Выведите в консоль реальный ответ API
2. Обновите интерфейсы в `lib/types.ts`
